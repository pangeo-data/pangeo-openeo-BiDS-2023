FAIR metadata generation

FAIR metadata generation#

FAIR principles provide some guidelines when producing and sharing datasets.

An openEO backend can make it easier to comply with these principles, by incorporating these principles in the implementation, so that users are automatically a step closer to generating FAIR-compliant open data.

These are a few examples that you want to look for as a user, or that a backend provider may want to implement:

  • F2 Rich metadata openEO backends generate STAC metadata, backends are encouraged to make it rich by including processing info, complete raster metadata, band information, etcetera.

  • R1.2 Detailed provenance In result metadata derived-from links can link back to all input products to provide provenance.

  • R1.3 use of domain relevant metadata standard openEO generates STAC metadata, so this one is included by default.

As openEO backends do not automatically publish the data you generate, thereâ€™s also a number of principles that still require some attention, but still, with data and metadata generated by openEO, you should get a headstart!
